Graph Similarity Computation

Introduction

With strong representative abilities, graphs have a broad range of applications in various fields, including social network study, computational chemistry, and biomedical image analysis. Recently, especially with the development of deep learning techniques, there aroused surging interests on graph-related problems.

As for my project experience, my research subject lies in graph similarity prediction problem, which is one of the fundamental challenges of many real-world applications such as biological molecular similarity search and social group network similarity identification.

To depict the similarity between two graphs, one of the pervasive metrics is GED (short for graph edit distance). Orthodox methods like A* could calculate exact GED values but require exponential time complexity. Some approximate methods include VJ, BEAM, HUNGRIAN can generate an estimated value of GED, however, those methods still have polynomial or even sub-exponential time complexity.

What we are trying to do is to deploy graph neural networks (GNNs) as a data-driven solution for this task. To date, a bunch of graph convolution techniques have been proposed such as graph convolution network (GCN), edge convolution network, Graph Isomorphism Network (GIN), graph attention network (GAN) which are proven to be powerful tools to embed node features. Also, to utilize pooling techniques in graph, researchers have developed several downsampling methods including diffpool, topkpooling, sagpool and mempool.

Back to the graph similarity computation task, this problem can be divided into two parts: (1) How to embed the graph? (2) With the feature vectors in hand, how to measure the similarity?

Existing deep learning methods for graph domain including SimGNN, Gsim_CNN, GMN, HGMN have done a wonderful job in this field. Those methods can be roughly divided into two categories: (1) directly map the graph into features, such as GCN-MEAN, GCN-MAX (2) embed a pair of graphs at a time to get cross-graph information, this methodology is featured by GMN, other methods including Gsim_cnn, SimGNN don’t exert pair embedding, but they still have some cross-graph mechanism to facilitate matching.

The first category of models can be called as embedding models, the second one is named as matching models. Obviously, the disadvantage of matching models is the relatively high time complexity, which is the cost of better accuracy. Furthermore, those matching techniques are not so efficient when employed in large graphs. So how can we make a trade-off between these two categories?

We have two solutions:

  1. Coarsen the graph to a smaller one
  2. Partition the graph to smaller ones

Simple and direct, so the problem becomes how to design a proper coarsening method and how to partition the graph to let the subgraph attain the information of the source graph as much as possible?

Those problems will be discussed later.
